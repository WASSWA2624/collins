Collins Version 5 deployment package

Folders:
- backend-v5/: cPanel/DirectAdmin Node.js application root.
- frontend-v5/: static Expo web export for the public web root.
- database-v5/: phpMyAdmin-ready MySQL/MariaDB SQL files.
- android-apk-v5/: signed Android release APK and Gradle output metadata.
- ios-ipk-v5/: iOS build status note. No iOS binary is included because EAS signing credentials are not configured for non-interactive cloud builds in this workspace.

Backend deployment:
1. Upload the contents of backend-v5/ to the Node.js app root, for example /home/zelahco/collins-backend.
2. Use Node.js 20 or newer.
3. Use cpanel-start.cjs as the startup file.
4. Review .env.production on the server before restart.
5. Run npm install from the backend app root.
6. Run npm run db:migrate:deploy from the backend app root, or import the matching SQL from database-v5/ through phpMyAdmin.
7. Restart the Node.js app and verify /ready and /api/v1/health.

Database deployment:
1. For a fresh empty database, import database-v5/collins-database-v5-full.sql through phpMyAdmin.
2. For an existing Version 4 database migrated through 20260511110000_patient_age_days, import database-v5/collins-database-v4-to-v5.sql.
3. Always back up an existing database before importing SQL through phpMyAdmin.
4. Version 5 includes all migrations through 20260511221000_six_character_app_codes.

Frontend deployment:
1. Upload the contents of frontend-v5/ to the web document root.
2. Keep .htaccess with the upload so client-side routes resolve to index.html.
3. The static web export uses Unix-style relative asset paths generated by Expo and is compatible with Linux/cPanel document roots.
4. The frontend build is configured for https://api.zelah.co.ug/api/v1 through the production env file.
5. Frontend production build number is 5.

Android APK:
1. Install android-apk-v5/collins-api-zelah-release-v5.apk on Android devices.
2. The APK is a signed release build for com.collins.android.
3. Android versionCode is 5 and versionName is 1.0.0.
4. The embedded bundle was built with the production env file.

iOS status:
1. Local iOS builds are not possible from this Windows workspace because there is no Xcode/iOS toolchain and no native ios/ directory.
2. EAS CLI is authenticated, but both preview and production iOS builds failed before build start because Apple signing credentials are not configured for non-interactive builds.
3. After configuring Apple credentials interactively, run an EAS iOS build and place the downloaded .ipa in ios-ipk-v5/.
4. The local app config has ios.bundleIdentifier=com.collins.ios and ios.buildNumber=5.

POSIX/cPanel compatibility:
- Deployment instructions use Unix/Linux paths such as /home/zelahco/collins-backend.
- The web package includes .htaccess rewrite rules for Apache/LiteSpeed cPanel hosting.
- Runtime scripts use Node.js entry points and avoid Windows-only shell wrappers in production startup.
- Artifact folder and zip names use lowercase POSIX-style hyphenated names.
